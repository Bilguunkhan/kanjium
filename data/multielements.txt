Notes:
* in the list below, for example, 屶 = 山 means that one should use 屶 to search the database for all kanji containing the kanmuri element 山, but the multielements lookup table of a website/app should list it as 山 for simplicity's sake.
* where there is a mention of "MultiElements.woff/.svg/.ttf", it means this custom font should be used to properly render this element on all devices and platforms.
* where there is a character in brackets, for example, 呉 (𠃑) and 忌 (心), it means the first character (e.g. 呉 and 忌) should be used to search the database and be listed and rendered on the multielements lookup table of a website/app with the custom font, whereas the character in brackets can be ignored and is here only as a visual internal memo.
* the element lookup database only uses 3-byte characters, as I found out 4-byte characters can be (or used to be at least, so this might be redundant nowadays) troublesome with various queries/regexes, so for example 𠆢 (U+201A2) is internally substituted with ㆟ (U+319F).
